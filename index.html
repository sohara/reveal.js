Benefits

  • Fewer lines of code
  • Fewer assignments => fewer opportunities for errors/bugs
  • Low-level implementations live in fewer places (hopefully just one)
  • Top-level UI becomes declarative instead of imperative.
  • Pushing implementation details to lower level, so that top levels are about declaring the composition.

How is this possible?

Through a mix of existing code re-use practices and some nifty API additions to Ember.

Contextual Components

Allows more structured composition of nested components

Will hopefully enable simpler communication patterns between nested components, while maintaining isolation and DDAU.

A "closing-over" component helper

Using the (component helper allows invocation of components with "closed over" (bound) properties.

Will likely later allow for local component lookups

A hash helper

In conjunction with yield or with it allows setting up new groups of components


              {{yield (hash input=(component "my-text-input" model=model))}}

Accessed and lookedup up via a dot-syntax


{{#form-for model=model as |f|}}
  {{f.input field="firstName"}}
{{/form-for}}

Basically a big hash




  {{publish-info model=model}}

    card-list=(component "content-form/card-list" model=model)

    tags=(component "content-form/tags-field" model=model)
    text-field=(component "content-form/text-field" model=model)
    textarea=(component "content-form/cf-textarea" model=model)
    thumbnail=(component "content-form/thumbnail-upload" model=model)
    )}}

Components get entire model

They typically also get a 'field' attr (string) to be looked up on the model.

This can be achieved dynamically using just templates (with new helpers)


{{textarea value=(mut (get model field))}}

{{validation-messages messages=(get model (concat 'validationErrors.' field '.messages'))}}

Simple Code re-use

More DRY-ness that can be achieve via things we can take more advantage of:

  • mixins
  • componentizing repeated stuff (showing validations)

Example: content-form-inputs mixin


import Ember from 'ember';
const { computed } = Ember;

export default Ember.Mixin.create({
  classNameBindings: 'gridClass',

  gridClass: computed('customGridClass', function () {
    return this.get('customGridClass') || this.get('defaultGridClass');
  }),

  labelText: computed('field', 'customLabel', function () {
    if (this.get('customLabel')) {
      return this.get('customLabel');
    }
    const field = this.get('field');
    return field && field
      .decamelize()
      .split('_')
      .join(' ')
      .capitalize();
  })
});
